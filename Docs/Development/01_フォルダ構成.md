# 01_フォルダ構成

このドキュメントでは、BitzBufferプロジェクトの主要なフォルダ構成と、それぞれのフォルダが持つ役割について説明します。

**推奨事項:** 各主要フォルダには、そのフォルダの目的や使い方を説明する `README.md` を配置することを推奨します。これにより、GitHub上でフォルダを閲覧した際に、内容を理解しやすくなります。

## ルートディレクトリ

プロジェクトのルートディレクトリ直下には、以下の主要なファイルやフォルダが存在します。

```
. /
 ├── .git/ # Gitの管理ファイル (通常は意識しない) 
 ├── .gitignore # Gitの追跡対象外ファイルを指定 
 ├── Docs/ # このドキュメント群を含む、プロジェクト関連ドキュメント 
 ├── Src/ # ライブラリのソースコード 
 ├── Sandbox/ # 機能試用やアイデア実験のための砂場 
 ├── Tests/ # テストコード 
 ├── README.md # プロジェクトの概要説明
 └── ...
```

## Docs/

プロジェクトに関するドキュメントを格納するフォルダです。主に以下のサブフォルダで構成されます。

```
Docs/
 ├── Development/ # 開発環境や開発ルールに関するドキュメント 
 │ ├── 00_はじめに.md 
 │ └── 01_フォルダ構成.md (このファイル) 
 │ └── ... (02_ブランチ戦略.md など) 
 ├── DesignSpecs/ # ライブラリの設計仕様書 
 │ ├── 00_Overview.md
 │ ├── 01_Core_Interfaces.md
 │ ├── 02_Providers_And_Buffers.md
 │ ├── 03_Pooling.md
 │ ├── 04_GPU_Support.md
 │ └── 05_Error_Handling.md
 └── ...
```

### Docs/Development/
この開発ドキュメント群が格納されています。開発環境の構築手順、コーディング規約、ブランチ戦略など、開発を進める上で必要な情報を集約します。

### Docs/DesignSpecs/
BitzBufferライブラリのコアとなる設計思想、API定義、アーキテクチャに関する詳細な仕様書が格納されています。`README.md` からもリンクされています。

## Src/

BitzBufferライブラリのC#ソースコードが格納されるメインフォルダです。
プロジェクトファイル (`.csproj`) や、各機能モジュールごとのサブフォルダなどが配置される予定です。

```
Src/
 ├── BitzBuffer/ # ライブラリ本体のプロジェクト (例) 
 ├── BitzBuffer.csproj 
 └── ... (ソースファイル群)
```

## Tests/

BitzBufferライブラリの単体テストや結合テストなどのテストコードを格納するフォルダです。
テストプロジェクト (`.csproj`) や、テスト対象に応じたサブフォルダなどが配置される予定です。

```
Tests/ 
 ├── BitzBuffer.Tests/ # テストプロジェクト (例) 
 └── BitzBuffer.Tests.csproj └── ... (テストコードファイル群)
```

## Sandbox/

機能試用やアイデア実験のための砂場のコードを格納するフォルダです。
本格的なベンチマークやテストとは異なり、気軽にコードを書いて試すことを目的としています。

```
Sandbox/ 
 ├── BitzBuffer.Sandbox/ # サンドボックスプロジェクト (例) 
 ├── BitzBuffer.Sandbox.csproj 
 ├── Program.cs
 └── ... (実験用コードファイル群)
```
