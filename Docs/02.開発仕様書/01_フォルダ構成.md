# 01_フォルダ構成

このドキュメントでは、BitzBufferプロジェクトの主要なフォルダ構成と、それぞれのフォルダが持つ役割について説明します。

**推奨事項:** 各主要フォルダには、そのフォルダの目的や使い方を説明する `README.md` を配置することを推奨します。これにより、GitHub上でフォルダを閲覧した際に、内容を理解しやすくなります。

## ルートディレクトリ

プロジェクトのルートディレクトリ直下には、以下の主要なファイルやフォルダが存在します。

```
. /
 ├── .git/                 # Gitの管理ファイル (通常は意識しない)
 ├── .gitignore            # Gitの追跡対象外ファイルを指定
 ├── Docs/                 # プロジェクト関連ドキュメント (このドキュメント群を含む)
 │   ├── 01.システム仕様書/    # 要求仕様にあたる内容
 │   ├── 02.開発仕様書/      # 開発ルールや環境に関するドキュメント (このファイルが属する)
 │   └── 03.詳細仕様書/      # 各機能の詳細な設計仕様書
 ├── Src/                  # ライブラリのソースコード
 ├── Sandbox/              # 機能試用やアイデア実験のための砂場
 ├── Tests/                # テストコード
 ├── README.md             # プロジェクトの概要説明 (ルート)
 └── ...                   # その他の設定ファイルなど
```

## Docs/

プロジェクトに関する全てのドキュメントを格納するメインフォルダです。以下の主要なサブフォルダで構成されます。

### Docs/01.システム仕様書/

プロジェクト全体の目的、背景、主要な機能要件、ターゲットユーザー、解決したい課題など、ライブラリの「なぜ作るのか」「何を作るのか」といった上位レベルの仕様を記述するドキュメント群を格納します。
（例: `00_プロジェクト概要と目的.md` など。具体的なファイルはプロジェクトの進行に応じて作成）

### Docs/02.開発仕様書/

このドキュメント（`01_フォルダ構成.md`）を含む、開発環境の構築手順、本ドキュメントで説明するフォルダ構成、ブランチ戦略、コーディングルールなど、BitzBufferプロジェクトの開発を進める上で必要な規約や情報を集約します。

*   **[`00_はじめに.md`](./00_はじめに.md)**: 開発仕様書全体の概要と目的。
*   **[`01_フォルダ構成.md`](./01_フォルダ構成.md)** (このファイル): プロジェクトの主要なフォルダとその役割。
*   **[`02_ブランチ戦略と命名規則.md`](./02_ブランチ戦略と命名規則.md)**: Gitのブランチ運用ルール。
*   **[`03_コーディングルール.md`](./03_コーディングルール.md)**: コードを書く上での規約。

### Docs/03.詳細仕様書/

BitzBufferライブラリの各主要コンポーネントや機能モジュールに関する詳細な設計思想、API定義、アーキテクチャに関する仕様書を格納します。

*   **`BitzBuffer/`**: コアとなるバッファ管理ライブラリ本体の設計仕様書群。
    *   **[`00_プロジェクト概要と目的.md`](../03.詳細仕様書/BitzBuffer/00_プロジェクト概要と目的.md)**: BitzBufferコアの概要。
    *   **[`01_コアインターフェース.md`](../03.詳細仕様書/BitzBuffer/01_コアインターフェース.md)**: コアインターフェースの詳細。
    *   *(他のBitzBufferコア設計仕様書へのリンクも同様)*
*   **`BitzBuffer.Pipelines/`**: 高レベルな非同期データ処理パイプライン機能の設計仕様書群。(内容は今後定義)
    *   **[`00_P_プロジェクト概要と目的`](../03.詳細仕様書/BitzBuffer.Pipelines/00_P_プロジェクト概要と目的.md)**: BitzBuffer.Pipelinesの概要。(今後作成)
    *   *(他のBitzBuffer.Pipelines設計仕様書へのリンクも同様)*

## Src/

BitzBufferライブラリのC#ソースコードが格納されるメインフォルダです。
プロジェクトファイル (`.csproj`) や、各機能モジュールごとのサブフォルダなどが配置される予定です。

```
Src/
 ├── BitzBuffer/             # ライブラリ本体のプロジェクト (例: BitzBuffer.Core.csproj)
 ├── BitzBuffer.Pipelines/   # パイプライン機能のプロジェクト (例: BitzBuffer.Pipelines.csproj)
 └── ...                     (ソリューションファイル .sln など)
```

## Tests/

BitzBufferライブラリの単体テストや結合テストなどのテストコードを格納するフォルダです。
テストプロジェクト (`.csproj`) や、テスト対象に応じたサブフォルダなどが配置される予定です。

```
Tests/
 ├── BitzBuffer.Tests/           # コア機能のテストプロジェクト
 ├── BitzBuffer.Pipelines.Tests/ # パイプライン機能のテストプロジェクト
 └── ...
```

## Sandbox/

機能試用やアイデア実験のための砂場のコードを格納するフォルダです。
本格的なベンチマークやテストとは異なり、気軽にコードを書いて試すことを目的としています。

```
Sandbox/
 ├── BitzBuffer.Sandbox/       # サンドボックスプロジェクト
 └── ...
```
