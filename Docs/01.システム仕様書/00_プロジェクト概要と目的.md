# BitzBuffer プロジェクト概要と目的

このドキュメントは、C# Buffer管理ライブラリ「BitzBuffer」プロジェクト全体の目的、スコープ、主要な機能要件、および関連プロジェクトについて記述します。これは、ライブラリ開発の指針となる最上位の仕様です。

## 1. プロジェクトの目的

BitzBufferプロジェクトの主要な目的は以下の通りです。

*   **高性能なバッファ管理の提供:** 画像処理、CAD/モデリング、機械学習用テンソル、FAプロトコル通信など、現代のアプリケーションで頻繁に発生する大量のデータセットを効率的に処理するための、高性能なバッファ管理基盤を提供します。
*   **多様なメモリ種別の統一的取り扱い:** マネージドメモリ、アンマネージドメモリ（ネイティブメモリ）、そして将来的にはGPUメモリといった異なる種類のメモリリソースを、統一されたプログラミングインターフェースを通じてシームレスに扱えるようにします。
*   **メモリ効率の最大化:** ガーベージコレクション (GC) の負荷軽減、ラージオブジェクトヒープ (LOH) の回避、メモリ断片化の抑制、そしてゼロコピー操作の促進を通じて、アプリケーションのメモリ効率と全体的なパフォーマンスを向上させます。
*   **開発者エクスペリエンスの向上:** 使いやすく、理解しやすいAPIを提供することで、開発者が複雑なメモリ管理の詳細を意識することなく、アプリケーションのコアロジックに集中できるように支援します。Fluent Interfaceの採用も検討します。
*   **学習と探求:** 最新の .NET のメモリ管理機能やパフォーマンス最適化技術（例: `Span<T>`, `Memory<T>`, `ReadOnlySequence<T>`, `MemoryManager<T>`, `SafeHandle`）の活用方法を深く探求し、その知見をライブラリの設計と実装に反映させます。これは開発者自身の技術力向上も目的の一つです。

## 2. プロジェクトのスコープ

*   **ターゲットフレームワーク:** .NET 6 以降を主要なターゲットとします。一部の高度な機能については、.NET 8 以降の利用も許容します。
*   **提供する主要機能:**
    *   バッファの確保（プーリングされたもの、直接生成されたもの）。
    *   バッファの解放とライフサイクル管理（`IDisposable` パターン、所有権管理）。
    *   効率的なバッファプーリング戦略（サイズ別バケット、カスタマイズ可能）。
    *   多様なデータ型 (`struct` 制約) への対応。
    *   マネージドメモリとアンマネージドメモリへの対応（拡張可能なプロバイダモデル）。
    *   非連続メモリ（セグメント化されたバッファ）のサポートと、`ReadOnlySequence<T>` との連携。
    *   ゼロコピーでのバッファセグメントのアタッチ（所有権移譲）機能。
    *   GPUメモリ管理のサポート（初期実装では拡張ポイントの提供に留め、具体的なAPI対応は将来の別モジュール）。
*   **スコープ外 (初期実装):**
    *   特定のファイルI/OやネットワークI/Oに特化した高レベルAPI（ただし、それらの基盤となるバッファ管理は提供）。
    *   `System.IO.Stream` との直接的な高度な連携機能（基本的なデータのやり取りは可能）。
    *   分散環境でのバッファ共有や同期。

## 3. 関連プロジェクト

*   **`BitzBuffer.Pipelines` (構想中):**
    *   BitzBufferコアライブラリのバッファ管理機能を基盤として、`System.IO.Pipelines` に似た、より柔軟で高性能な非同期データ処理パイプライン機能を提供する別プロジェクト（またはモジュール）です。
    *   スレッド間、プロセス間、ネットワーク間など、多様な通信スコープに対応することを目指します。
    *   このプロジェクトの詳細は、別途 `Docs/03.詳細仕様書/BitzBuffer.Pipelines/` 配下の設計仕様書で定義されます。
